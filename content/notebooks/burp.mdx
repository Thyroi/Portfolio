## Core idea

Use Burp when you need visibility into session flow, headers, and subtle server
responses that scanners flatten away.

## Useful reminders

- Create a clean project config for each target.
- Keep repeater tabs named by hypothesis.
- Save successful requests before mutating them further.

## Common flow

```text
Proxy -> Target -> Repeater -> Comparer
```
