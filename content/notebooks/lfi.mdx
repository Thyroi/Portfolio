# Local File Inclusion (LFI) — Guía rápida

Resumen: LFI (Local File Inclusion) es una vulnerabilidad en aplicaciones web que permite a un atacante leer (y en algunos casos ejecutar) ficheros locales del servidor mediante la inclusión no segura de rutas de ficheros en funciones del lenguaje (por ejemplo `include`, `require` en PHP). Se explota típicamente a través de parámetros que controlan rutas a ficheros.

**Qué es y cómo funciona**

- Definición: LFI permite que una aplicación haga referencia a un fichero local controlado por el atacante o a una ruta arbitraria en el sistema de ficheros.
- Vector típico: un parámetro URL/POST que se pasa a una función de inclusión (PHP: `include($_GET['page'])`) o que el servidor usa para abrir y mostrar contenido.
- Diferencia con RFI: RFI (Remote File Inclusion) permite incluir archivos remotos (por URL); LFI se limita a ficheros locales, pero puede llevar a RCE con técnicas adicionales (log poisoning, wrappers, uploads).

**Qué tener en cuenta antes de probar**

- Solo en entornos autorizados (laboratorio/CTF/permiso explícito).
- Revisar versión de servidor y stack (PHP, Apache, módulos habilitados) y permisos de ficheros.
- Si el servidor corre en Windows, la sintaxis de rutas cambia (`C:\path\file`) y no siempre permiten técnicas Unix-like.

**Técnicas y vectores comunes**

- Path traversal: `?file=../../../../etc/passwd`
- Encoding/normalización: `..%2f..%2f`, `%2e%2e%2f`, `..\..\` (Windows)
- Null byte (histórico): agregar `%00` para truncar cadenas en binarios antiguos (menos efectivo hoy)
- Wrappers PHP: `php://filter/read=convert.base64-encode/resource=...` para leer ficheros binarios o forzar encoding
- Log poisoning: inyectar payloads en logs (User-Agent, Referer) luego incluir `/var/log/apache2/access.log` para ejecutar código si logs interpretan como PHP
- Upload + include: subir un archivo `.php` si existe upload y luego incluirlo

**Detección (enumeración manual y automática)**

- Inspeccionar formularios y parámetros que cargan páginas (p.ej. `?page=`, `?lang=`, `?template=`).
- Peticiones probando traversal y wrappers; guardar respuestas y tamaños.
- Herramientas: `ffuf/gobuster` para descubrir rutas, `Burp` para pruebas manuales, `curl` para pruebas rápidas.
- Pruebas útiles (ejemplos):
  - `curl 'http://target/?page=../../../../etc/passwd'`
  - `curl 'http://target/?page=php://filter/read=convert.base64-encode/resource=index.php'`

**Elevación desde LFI a RCE (escenarios comunes)**

- Log poisoning + `php` interpretation: si los logs son incluidos y están escritos con código PHP inyectado, puede ejecutarse.
- Uploadable files: subir un `.php` y luego incluirlo.
- Inclusion de sockets o dispositivos (en sistemas mal configurados) — depende de permisos.
- Notas: muchas de estas técnicas requieren condiciones adicionales (permisos, módulos, configuración `allow_url_include`, etc.).

**Mitigaciones y buenas prácticas**

- No usar `include` con entradas directamente controladas por el usuario; validar y normalizar rutas.
- Usar listas blancas (whitelists) de ficheros permitidos en vez de concatenar rutas.
- Restringir permisos de ficheros (el usuario del servidor web no debe poder leer ficheros sensibles).
- Desactivar `allow_url_include` y revisar `open_basedir` en PHP.
- Separar código y datos; evitar ejecutar código desde ubicaciones donde se almacenan uploads o logs.
- Hacer hardening del servidor (actualizaciones, módulos innecesarios deshabilitados).

**Cómo leer un CVE (pasos y campos clave)**

1. Identificador: `CVE-YYYY-NNNNN` — único.
2. Descripción: resumen técnico del problema (qué hace, condiciones de explotación).
3. Affected versions / CPE: versiones afectadas (p.ej. "&lt;= 2.4.51" o "2.4.49 and 2.4.50").
4. CVSS: puntaje de severidad (3.x o 4.0) y vector (AV/AC/PR/UI/C/I/A) que explica el contexto de explotación.
5. CWE: clase del fallo (ej. `CWE-22 Path Traversal`, `CWE-79 XSS`, `CWE-787 OOB write`).
6. References: advisories, parches, PoC, listas de correo — revisa vendor advisory primero.
7. Mitigación/patch: versión donde se corrige y recomendaciones.
8. Timeline / notas: fecha de publicación, modificaciones posteriores.

**Ejemplos (CVEs relacionadas con Apache 2.4.52 según el advisory oficial)**

- `CVE-2021-44790` (mod_lua multipart parser):
  - Descripción: buffer overflow en `mod_lua` al parsear cuerpos multipart (r:parsebody()).
  - Afecta: Apache httpd 2.4.51 y anteriores (NVD: "Up to (excluding) 2.4.52").
  - Severidad (NVD): CVSS 3.x — 9.8 (CRITICAL) según la página NVD.
  - CWE: `CWE-787` (Out-of-bounds Write).
  - Relevancia: si el servidor usa `mod_lua` y el sitio procesa multipart por Lua, es crítico. Para LFI esto no es LFI puro, pero muestra cómo un módulo del servidor puede introducir ejecución remota.

- `CVE-2021-44224` (forward proxy SSRF / NULL deref):
  - Descripción: una URI mal formada enviada a httpd configurado como forward proxy puede causar crash (NULL deref) o, en ciertas configuraciones, permitir SSRF via encaminamiento a sockets/UNC.
  - Afecta: 2.4.7 up to 2.4.51 (NVD: up to excluding 2.4.52). NVD base score: 8.2 (HIGH).
  - CWE: `CWE-476` (NULL Pointer Dereference).
  - Relevancia: afecta configuraciones proxy; no es LFI, pero demuestra cómo la versión del servidor define la exposición a fallos graves.

**Cómo usar esta info en tu laboratorio (seguro)**

- Identifica si el servidor objetivo usa módulos (PHP, mod_lua, mod_proxy). Si no, varios CVE no son aplicables.
- Para aprender LFI: crea una pequeña app PHP en local con `include $_GET['p']` y prueba traversal, `php://filter`, y mitigaciones (whitelist y realpath checks).
- Revisa el `Vendor Advisory` (en la página Apache) para ver la versión que corrige el fallo y el parche aplicado.

## LFI en Windows — consideraciones y payloads

- Rutas típicas en Windows que puedes intentar si el servidor es Windows (ajusta mayúsculas/minúsculas):
  - `C:\Windows\System32\drivers\etc\hosts`
  - `C:\Windows\win.ini`
  - `C:\Apache24\conf\httpd.conf` (o `C:\Apache\conf\httpd.conf`) si Apache está instalado en `C:\Apache24`
  - `C:\inetpub\wwwroot\web.config` (si hay IIS mezclado o uploads)

- Payload traversal Windows (ejemplos):
  - `../../../../../../../../Windows/System32/drivers/etc/hosts`
  - `..\..\..\..\..\..\Windows\System32\drivers\etc\hosts`
  - `../../../../..//..//..//windows/win.ini`

- Wrappers y técnicas aplicables en Windows también:
  - `php://filter/read=convert.base64-encode/resource=C:/path/to/file.php` — útil para recuperar el código fuente PHP cuando la inclusión devuelve contenido no interpretado.
  - Log poisoning: en Windows busca rutas de logs de Apache (`C:\Apache24\logs\access.log`) y de IIS (`C:\inetpub\logs\LogFiles\...`) y prueba inyectar User-Agent con payload PHP y luego incluir el fichero de log.

- Notas sobre diferencias con Unix:
  - Separador de rutas (`\` vs `/`) y diferentes ubicaciones de ficheros.
  - Permisos y cuentas: en Windows el usuario del servicio web puede leer distintos ficheros que en Linux; probar rutas comunes es clave.
  - Técnicas como `php://filter` funcionan igual en plataformas que ejecutan PHP.

## Comandos rápidos (ejemplos) — ajusta `Host`/IP

```bash
# barra normal (Unix-style traversal hacia path Windows)
curl -s -H 'Host: unika.htb' "http://10.129.95.234/?page=../../../../../../../../Windows/System32/drivers/etc/hosts" -i

# php://filter para leer fuente PHP (base64)
curl -s -H 'Host: unika.htb' "http://10.129.95.234/?page=php://filter/read=convert.base64-encode/resource=/var/www/html/index.php" | base64 -d

# ffuf para fuzzear parametro page con payloads (usa lista local `lfi_payloads.txt`)
ffuf -u 'http://unika.htb/?page=FUZZ' -w lfi_payloads.txt -H 'Host: unika.htb' -mc all -t 40 -o responder_ffuf_lfi.json
```

## Pistas para identificar inclusión en respuestas

- Diferencias de tamaño/respuesta cuando inyectas payloads.
- Mensajes de error que revelen rutas o permisos.
- Respuestas con contenido no esperable (por ejemplo un hosts o win.ini mostrado).

**Advertencia legal y ética**

- No pruebes estas vulnerabilidades contra sistemas sin permiso explícito.
- Emplea máquinas de práctica (HTB, VMs, contenedores) o entornos controlados.
